# Product Notes

The product should be framed as recruiting decision support. It helps structure review, but human accountability remains central.

## User Journeys

- Candidate completes profile and answers prompts
- Recruiter reviews ranked or scored candidates
- Maintainer adjusts platform settings or review criteria

## Functional Requirements

- Support candidate, recruiter, and maintainer roles
- Capture candidate data and answers
- Produce review signals that can be explained

## Constraints

- Hiring tools can create fairness and compliance risk
- Scoring must be validated against real-world outcomes before trust
- Sensitive candidate data requires strong privacy controls

## Roadmap Ideas

- Add documentation for scoring methods
- Add fairness and bias evaluation notes
- Add tests around role permissions and scoring boundaries
